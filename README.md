# mainn.c


----------------------------------------------------------------------------------------------------------
1.top 명령어: 리눅스 운영체제에세사용하는 시스템 모니터링, 프로세스 관리도구
----------------------------------------------------------------------------------------------------------
2. 특징
-실시간 모니터링: 설정된 주기마다 화면을 갱신하여 시스템을 파악
-리소스 점유율: 리소스 점유율을 파악하여 어떠한 프로세스가 cpu나 메모리를 과도하게 점유하여 시스템 속도에 영향을 미치는지 확인할 수 있다
-프로세스 제어:프로세스를 종료하거나 우선순위를 변경하는 등의 제어가 가능하다
----------------------------------------------------------------------------------------------------------
3. 화면: top을 실행하면 상단의 "시스템 요약 정보"와 하단의 "프로세스 목록"으로 나뉜다

3.1 시스템 요약 정보:시스템 가동시간, 평균부하, 프로세스 상태,cpu 사용률,메모리 사용량 정보를 보여준다 
-시스템 가동시간: 시스템이 켜진 후 얼마나 지났는지, 현재 접속자 수는 몇 명인지 표시한다
-평균 부하: cpu가 처리해야 할 작업의 대기열 길이를 의미한다
-프로세스 상태: 전체 프로세스의 개수와 상태별 분류를 보여준다(Running,Sleeping,Stopped,Zombie)
-cpu 사용률: cpu가 어디에서 많이 사용하고 있는지를 백분율로 보여준다
-메모리 사용량: 메모리 사용량을 표시한다

3.2 프로세스 목록: 개별 프로세스들의 상세정보를 보여준다
-PID : Process ID, 프로세스의 고유 식별 번호
-USER: User Name, 해당 프로세스를 실행한 사용자 계정 이름
-PR: Priority, 프로세스의 실행 우선순위
-NI: Nice Value, 사용자가 조절 가능한 우선순위 값
-VIRT: Virtual Image, 프로세스가 사용 중인 가상 메모리 총량
-RES: Resident Size, 프로세스가 사용 중인 실제 물리 메모리 양
-%CPU: CPU 사용률
-%MEM: 메모리 점유율
-COMMAND: 해당 프로세스를 실행한 명령어 이름
---------------------------------------------------------------------------------------------------------
4. 주요 조작키
P: cpu 사용률이 높은 순서대로 정렬
M: 메모리 점유율이 높은 순서대로 정렬
k: 특정 프로세스를 강제 종료
q: top 프로그램 종료

1. ps: linux 시스템에서 실행중인 프로세스들의 스냅샷을 출력하는 도구
--------------------------------------------------------------------------------------------------------
2. 특징
-정적 상태확인: top과 달리 시스템의 상태를 실시간으로 보여주지 않고 ps명령어를 입력한 시점을 출력한다
-프로세스 계층 파악: PPID를 확인할 수 있다. 어떤 프로그램이 어떤 프로그램을 실행시켰는지 상속관계 확인가능
--------------------------------------------------------------------------------------------------------
3. 화면: ps -ef를 입력하였을때 출력되는 항목
-UID: User ID, 프로세스를 실행시킨 사용자 ID
-PID: Process ID, 프로세스 고유 식별 번호
-PPID:Parent Process ID, 이 프로세스를 실행시킨 상위 프로세스의 PID
-C:cpu 사용률 기반의 factor값
-STIME: Start Time, 프로세스가 시작된 시간
-TTY: Teletypewriter, 프로세스가 실행된 터미널 번호
-TIME: 프로세스가 사용한 총 cpu 시간 합계
-CMD: 실행된 명령어, 인수를 포함한 전체 이름
--------------------------------------------------------------------------------------------------------
4. 주요 옵션
4.1 ps -ef
-e: 모든 프로세스를 출력
-f: 풀 포멧으로 상세정보를 출력

4.2: ps aux
-a: 다른 사용자의 프로세스까지 모두 표시
-u: 사용자 중심으로 리소스 사용률등을 표시
-x: 터미널 제어 없이 실행되는 프로세스 표시






------------------------------------------------------------------------------------------------------- 
1.jobs: 현재 사용자가 사용중인 shell 내에서 실행중인 작업들의 상태를 확인하는 도구
--------------------------------------------------------------------------------------------------------
2. 특징
- 세션 중심 관리: 시스템 전체가 아닌 현재 터미널 창에서 실행한 작업 목록만 보여줌
- 작업 상태 확인: 작업이 현재 백그라운드에서 실행중인지 일시정지 상태인지 쉽게 파악가능
- Job ID 사용: 복잡한 PID 대신 Job ID를 사용하여 작업을 제어할 수 있다
--------------------------------------------------------------------------------------------------------
3. 화면: jobs를 입력했을때
-Job ID: Job Identifier, 현재 쉘에서 관리하는 작업의 고유 번호
-기호: + --> 가장 최근에 접근한 작업 , -  ---> 두번째로 최근에 접근한 작업
-state: 작업의 현재상태(Running,Stopped,Done)
--------------------------------------------------------------------------------------------------------
4. 관련 제어 명령
-&: 명령어 뒤에 붙여서 실행하면 처음부터 백그라운드 모드로 실행한다
-Ctrl + Z: 현재 화면에서 실행중인 작업을 일시정지 시키고 백그라운드로 보낸다 
-bg: 일시 정지된 작업을 백그라운드에서 다시 실행되도록 한다
-fg: 백그라운드에 있는 작업을 다시 화면 앞으로 가져온다

1.kill: linux 시스템에서 실행중인 프로세스에 signal 를 보내는 명령어이다
----------------------------------------------------------------------------------------------------------
2. 특징
-signal기반 제어: signal를 전송하여 프로세스가 스스로 행도을 결정하게 유도한다
-PID 필수 사용: 고유한 식별 번호인 PID를 사용하여 정확히 원하는 프로세스만을 타깃으로 지정한다
----------------------------------------------------------------------------------------------------------
3. 주요 signal
-15: SIGTERM, 정상종료한다
-9: SIGKILL, 입력즉시 메모리에서 제거한다
-2: SIGINT, 실행중인 작업을 취소할때 사용
-1:SIGHUP, 기본 동작은 종료하며, 터미널 연결이 끊겼음을 알리는 신호
